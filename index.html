Log Drive



Visão geral do sistema

Esse documento tem o objetivo de descrever e declarar todos os propósitos e parâmetros para o desenvolvimento do aplicativo de gerenciamento de viagens e manutenção veicular, chamado LogDrive, que inicialmente será desenvolvido para Web e posteriormente recebendo versões para Android e iOS.

Análise de viabilidade

Esse documento descreve a viabilidade de implementação do projeto de diário de bordo, i.e. LogDrive. O objetivo é desenvolver um aplicativo que permita aos motoristas registrarem e monitorarem informações detalhadas sobre suas viagens, desempenho do veículo, consumo de combustível e manutenções.

O business case é um documento para apresentar uma ideia de projeto. Seu objetivo principal é convencer gerente, líder de departamento, RH, gestor, cliente ou qualquer outro tomador de decisões a aprovar a ideia apresentada.

Plano metodológico

Esse documento descreve o plano metodológico que será seguido no desenvolvimento do projeto LogDrive, relativo à disciplina Projeto e Desenvolvimento de Sistemas 1. Nesse documento, serão expostos fatores levados em conta no planejamento, como recursos de hardware e software, gerenciamento do pessoal da equipe, divisão de horário de trabalho, detalhes dos módulos, assim também como uma estimativa do cronograma e restrições.

Documento de requisitos

Documento que contém uma declaração oficial de o que os desenvolvedores do sistema devem implementar.

Especificação de requisitos

É um documento funcional, ou seja, ele descreve as funcionalidades que um sistema deve ter. No caso de uso é apresentado como o sistema deve reagir ao conjunto de entradas que receber, qual o comportamento desse sistema e as saídas que serão geradas. Uma especificação de caso de uso envolve, por exemplo, cálculos que o sistema deve executar, manipulação de dados entre outras informações.

Protótipo de baixa fidelidade

A intenção deste protótipo é definir as premissas de design básicas que irão reger o projeto daqui para frente. São escolhas de cores e de UX num geral para definir o fluxo de uso do aplicativo por parte no usuário.
